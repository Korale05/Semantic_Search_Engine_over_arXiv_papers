The Actual Roadmap (follow this strictly)
STEP 1 — Build the simplest search (Baseline)

Goal: Get something working fast

What you do:
Get dataset (arXiv)
Store:
    title
    abstract
Use:
    TF-IDF (sklearn)
Search:
    Input query → return top 5 papers
Output:
    A simple function:

search("deep learning for NLP")


Why this matters:

This becomes your baseline (BM25/TF-IDF)
Without this → you can’t prove improvement later

STEP 2 — Add Embeddings (Your strength)

Now use what you already know.

Use:
    sentence-transformers/all-MiniLM-L6-v2
What changes:
    Convert all abstracts → embeddings
    Store in FAISS
    Query → embedding → nearest neighbors
Now your system becomes:

👉 Semantic search (not keyword search)

STEP 3 — Add Re-ranking (THIS is what makes it FAANG-level)

This is where most students fail.

Flow:
    FAISS returns top 20 results
    Re-ranker (cross-encoder) scores them
    Sort again
Model:
    cross-encoder/ms-marco-MiniLM-L-6-v2
Why this matters:
    FAISS = fast but approximate
    Cross-encoder = slow but accurate

👉 Together = production system

STEP 4 — Evaluation (this is what makes it “real engineer work”)

You don’t just say “it works”

You prove:

    MRR (Mean Reciprocal Rank)
    NDCG

Compare:

    TF-IDF
    Embedding
    Embedding + Re-ranking

👉 THIS is what interviewers care about
